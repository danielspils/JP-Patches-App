## What already works (Phase 1)

- 16-slot patch list per bank (C1–C16, D1–D16) in left panel. Click-to-name; names persist to `library.json`.
- Full PG-200 panel rendered from `panel.svg`, injected into `#panel-host` at startup.
- **24 knobs** divided into:
  - 6 **snap knobs** (in `SNAP_ANGLES`): click cycles to next enum value. `cursor: pointer`. Params: `dco1_range`, `dco1_waveform`, `dco2_range`, `dco2_waveform`, `dco2_crossmod`, `lfo_waveform`.
  - 18 **smooth knobs**: vertical drag, 2°/1px sensitivity. `cursor: ns-resize`. Range −140° to +140°.
- 8 toggle **switches** (Freq Mod LFO/ENV pairs, VCA Mode, Chorus, DCO/VCF env polarity). All use the canonical 3-segment white/grey/dark style. Click anywhere on body cycles state.
- 4 **buttons** (Save / Load / Manual / Write). LEDs dark by default, light on mousedown.
  - **Save** writes a JSON library file via file-dialog.
  - **Load** reads `.wav` (shells out to `jx3p wav-to-json`) or `.json` (direct).
  - **Manual / Write** are visual-only — wired for Phase 3 (MIDI) and v1 tape-dump-write respectively.
- Event delegation: single `mousedown` on the SVG dispatches by `data-control="knob|switch|button"`.
- All 32 patch params reflected dynamically in the SVG when a patch is selected.

## Conventions

1. **Plain JS only.** No TypeScript, no React/Vue/Svelte. Direct DOM manipulation in `app.js`.
2. **Don't modify `panel.svg`** unless the change is essential and unavoidable. `panel_locked_v4.svg` is the current canonical reference snapshot. The panel is meant to be visually frozen; functional changes happen in `app.js` (tagging, event handling) without touching the SVG.
3. **Window stays at 1140×710** (`resizable: false`). UI changes must fit. Daniel's screen is only 1147×719 — anything wider will be clipped offscreen.
4. **CSP**: `default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'`. Update meta tag in `index.html` if external resources or `eval`-style features become needed.
5. **macOS-only for v1.** Code freely uses `~/`, `pkill`, etc.
6. **Commit messages** typically prefixed with `feat:`, `fix:`, `Spec:`, or topic. See `git log --oneline` for the style.
7. **Co-author trailer** on commits: `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>`.

## Architecture

### IPC surface (`preload.js` → `window.api.*`)

```js
loadPatches:  ()      → load ~/Desktop/patches.json
loadLibrary:  ()      → load ~/Library/.../jp-patches/library.json
saveLibrary:  (data)  → write same
loadPanelSvg: ()      → read renderer/panel.svg
tapeSave:     (data)  → file-dialog → write JSON
tapeLoad:     ()      → file-dialog → read .wav (jx3p decode) OR .json
```

All handlers live in `main.js`. New IPC handlers should follow the same pattern.

### Key tables in `app.js`

- `DISCRETE` — enum values per discrete param (mirrors upstream `jx3p/patch.py`)
- `SNAP_ANGLES` — visual marker angles for snap knobs; overrides default −140/+140 spread in `paramToAngle`
- `SNAP_CYCLE` — overrides DISCRETE for click-cycle on knobs with fewer visual markers than enum values (currently only `lfo_waveform`: skips `fast random`)
- `KNOB_REGISTRY` — maps SVG location (circle cx/cy or `<g translate(...)>`) to param name
- `SWITCH_REGISTRY` — 8 switches with body selector + type
- `BUTTON_REGISTRY` — 4 buttons with body selector + LED color

### Patch data shape (32 params per slot)

See `jx3p/patch.py` upstream for canonical types. Key gotchas:
- `dco2_waveform` has 4 values incl. `"noise"` — but Series Circuits MIDI kit can't transmit noise.
- `mystery` (uint8 0-15) is preserved on disk for round-trip integrity but never sent over MIDI.
- Continuous params are uint8 (0–255). The MIDI kit transmits 7-bit, so a round-trip loses the bottom bit.

## Common pitfalls

1. **`node_modules/` symlinks can vanish** after some npm operations. If `npm start` fails with `electron: command not found`, run `npm install` to restore the `.bin/` symlinks.
2. **`~/Desktop/patches.json` is hard-coded** as the boot-time patch source. A fresh user without this file sees an empty/error state. v1 polish item.
3. **The `jx3p` Python tool path is hard-coded** to `~/JP-Patches`. Same caveat — v1 polish or Phase 4 fix.
4. **macOS filesystem is case-insensitive.** `~/JP-Patches` and `~/jp-patches` are the same directory. Don't try to create a sibling clone with only-case differences.
5. **Don't commit `node_modules/`.** The first commit in repo history did, and required a `git filter-branch` rewrite to undo (GitHub rejected the push due to Electron Framework being >100 MB). The `.gitignore` handles this now but stay vigilant.
6. **Window can't grow past 1140×710** on Daniel's machine — his logical display is 1147×719. Anything wider will be partially offscreen.
7. **Knobs cap at ±140° rotation.** The `paramToAngle` math expects continuous params in 0–255 and discrete params indexed into `DISCRETE[param]`. Always clamp before applying.
8. **Loop suppression matters in Phase 3.** When MIDI lands, inbound CC must not trigger outbound CC. See spec §3.5.1.
